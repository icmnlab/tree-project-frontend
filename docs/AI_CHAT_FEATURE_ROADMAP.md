### 3. 資料庫架構 ✅
```sql
-- chat_logs 表 (2025.12 更新：支援多會話)
CREATE TABLE chat_logs (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(255) NOT NULL,    -- 按用戶區分
  session_id VARCHAR(50),           -- 對話會話 ID (新增)
  message TEXT NOT NULL,
  response TEXT NOT NULL,
  model_used VARCHAR(100),
  project_areas TEXT,
  message_type VARCHAR(20) DEFAULT 'pair', -- pair/user/assistant (新增)
  created_at TIMESTAMP DEFAULT NOW()
);
-- 索引
CREATE INDEX idx_chat_logs_session_id ON chat_logs(session_id);
CREATE INDEX idx_chat_logs_user_session ON chat_logs(user_id, session_id);
```

### 4. 對話紀錄機制 ✅
- **用戶區分**：根據 `user_id` 區分不同用戶的對話
- **會話追蹤**：使用 `session_id` 區分同一用戶的不同對話 (新增)
- **上下文窗口**：最近 30 分鐘內的 30 筆對話（同會話）
- **定期清理**：24 小時後自動刪除（每小時執行 + 啟動時執行）
- **本地持久化**：對話歷史儲存在 SharedPreferences (最近 7 天/10 個對話)

## 🚀 建議新增功能

### 優先級 1：高價值、易實現

#### 1.1 📊 查詢結果視覺化
- **功能**：將統計結果自動生成圖表
- **適用場景**：
  - 「各區位的樹木數量」→ 長條圖
  - 「碳儲存量分布」→ 圓餅圖
  - 「樹高與胸徑關係」→ 散點圖
- **技術**：使用 fl_chart 套件
- **用戶價值**：★★★★★（直觀理解數據）

#### 1.2 📝 查詢歷史記錄
- **功能**：保存常用查詢，一鍵重複執行
- **適用場景**：
  - 「上次查的花蓮港樹木」
  - 常用查詢快捷鍵
- **技術**：本地 SQLite + SharedPreferences
- **用戶價值**：★★★★☆（提升效率）

#### 1.3 🌳 樹木名片生成
- **功能**：為特定樹木生成精美的資訊卡片
- **適用場景**：
  - 查詢「編號7的樹」後，可生成分享用的名片
  - 包含：樹種、尺寸、碳儲存、健康狀態、位置
- **技術**：Flutter 自定義 Widget + 截圖分享
- **用戶價值**：★★★★☆（方便分享、報告使用）

### 優先級 2：中等價值

#### 2.1 🤖 AI 主動建議
- **功能**：根據查詢結果，AI 主動提供後續建議
- **範例**：
  - 查詢大樹後：「這些大樹碳儲存量高，建議優先保護」
  - 查詢需關注的樹：「發現 5 棵樹需要關注，建議安排檢查」
- **技術**：在 AI 回應後追加分析 prompt
- **用戶價值**：★★★★☆

#### 2.2 📈 碳匯趨勢分析
- **功能**：分析區域碳儲存趨勢
- **適用場景**：
  - 「高雄港的碳儲存趨勢」
  - 「預測明年的碳吸存量」
- **技術**：基於現有數據 + AI 分析
- **用戶價值**：★★★☆☆

#### 2.3 🔔 異常數據警示
- **功能**：當查詢發現異常數據時主動提醒
- **範例**：
  - 「發現 3 棵樹的碳儲存量為空，可能需要補測」
  - 「編號 45 的樹高異常偏低，建議複查」
- **用戶價值**：★★★★☆

### 優先級 3：進階功能

#### 3.1 🗣️ 語音輸入
- **功能**：支援語音轉文字輸入查詢
- **技術**：speech_to_text 套件
- **用戶價值**：★★★☆☆（現場調查時方便）

#### 3.2 📸 圖片識別（VLM）
- **功能**：上傳樹木照片，AI 識別樹種
- **技術**：SiliconFlow VLM API (Qwen3-VL)
- **用戶價值**：★★★★☆（實用但需額外開發）

#### 3.3 📊 自定義報告生成
- **功能**：根據對話內容生成 PDF 報告
- **技術**：pdf 套件 + AI 摘要
- **用戶價值**：★★★★★（報告需求高）
