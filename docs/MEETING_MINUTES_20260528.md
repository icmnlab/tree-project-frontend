## 一、會議摘要

1. **現場逐棵測量（現場連線）** 必須穩定、無 BUG；相較批次匯入，此流程較直覺且無需兩人同步工作，故無 ID 對齊的難題（該議題尚未有定案，時程上可能來不及完善）。
2. **批次藍牙匯入**暫時保留，但不投入主要開發；專注把逐棵模式與 SOP 做穩。
3. **維護量測、歷史紀錄**為第二階段；介面方向已有共識，細節待原型後再確認。
4. **資料階層名詞**需對齊：現行 APP「區位 → 改稱專案」「專案 → 改稱區（Block）」。
5. **部署**：移設圖資中心電腦（分空間、開 Port）；維護上為 push 到 GitHub 後自動部署。
6. **會後交付**：書面 SOP + 本會議紀錄。
7. **GPS 定案（會後）**：外接 GNSS 模組（LG290P）**不採購**；現場座標一律使用**手機定位**，且**不提供**「樹木位置／測站」選擇，**一律樹木位置**。

## 二、現場測量

### 2.1 現場作業模式（方向確認，ID 對齊仍待解）

- 調查員通常**兩人一組**：一人測樹高（測距儀），一人測胸高直徑 DBH（依規範量周長換算，**現場手動輸入**）。
- **DBH 不可**僅以測距儀測寬度取代。
- 每棵樹需 **GPS 座標**；**無 GPS 應移除或要求重測**。
- **初勘不需導航**；導航留給數月後回測／維護。
- 測距儀五位數 ID 與 APP 內部 ID／DBH 負責人對齊；在批次＋兩人分開量 DBH 情境下尤其複雜。
- 與會共識傾向：**逐棵現場連線**可讓「量完即填、即提交」，較能避開批次對齊問題。
- ID 對齊的完整邏輯尚未定案；考量離校時程，可能來不及在畢業前完善，需於 SOP 中標註現階段假設與限制。

### 2.2 APP 操作流程（現場連線）

1. 進入**現場測量 → 現場連線**，掃描藍牙並配對測距儀。
2. 第一棵量測前設定**專案／區**、**GPS 取點一律預設樹木位置**（不再提供 GPS 取點位置選擇）、**場次名稱**（整場共用）。
3. 每一棵固定順序：**儀器 SEND → APP 收到 → 手機 GPS 定位 → 填寫整合表單 → 提交**。
4. 提交後回到連線畫面，可繼續下一棵；換專案或換區時，則跳出連線，重新在場次設定調整。（註：2026-06 實作改為連線中可切換，已完成樹木不搬移，見 SOP §4.6。）

### 2.3 會議中發現之 BUG

**症狀**：第一棵完成填表並成功寫入後，儀器再 SEND 第二棵時，未跳出 GPS 定位畫面，也未進入整合輸入頁，流程卡住。

**處理狀態（2026-06）**：已修復 PHGF 封包黏接、處理鎖、NUS TX 優先訂閱；電腦端 12 棵連續 SEND 驗證通過。

## 三、批次匯入（暫緩）

- 可一次匯入多筆且儀器端帶 GPS，但流程複雜、BUG 風險高。
- **決議**：先保留、不擴充。
- 若誤按造成缺 GPS 後的重測會有 ID 錯序問題，曾討論以時間戳輔助對齊，目前尚無定案。

## 四、專案、區（Block）— 名詞對齊

### 4.1 會中討論

| 層級 | 說明 | 範例 |
|------|------|------|
| **專案** | 計畫層級（可有計畫編號） | 某濱海調查計畫 |
| **區（Block）** | 專案下的樣區，各有邊界；一專案可含多個區 | A 樣區、B 樣區 |

- 學生測量時：**先選專案 → 再選區**。
- 輸入樹木時以手動先選專案 → 再選區為主，並在最後正式匯入資料庫前二次確認。
- 邊界：新建區時上傳座標點檔優於手機手繪（兩種方式仍並存）；格式待洪老師提供範例。
- 整區資料查詢／匯出：APP 現可部分做到；研究者跨區統計、批次匯出待後續另開會議。

## 五、維護量測（討論中 → 2026-06 已實作）

### 5.1 構想

- 待測量任務改為**維護**入口：選要進行維護的區後，進入該區樹木的維護列表／地圖。
- 黃同學建議：區的選擇以地圖（或圖）上點選區／點選樹，再進入詳細量測或修正的清單或頁面。
- 量測方式與初測相同：逐棵儀器 SEND → 表單；寫入**新一次**量測紀錄。
- 需支援**新增樹木**（維護過程中發現新株）。

## 六、歷史紀錄（討論中 → 2026-06 已實作）

### 6.1 會中達成共識

- 每棵樹保留**所有歷次量測**；欄位至少：日期、樹高、胸徑（DBH）；可加備註等。
- **呈現位置**：
  - 表單頁之待測量任務欄下方：維護人員量測時可對照歷史（最新在上）。
  - 一般樹木詳情頁下方：一般使用者點樹亦可看歷史（非僅維護模式）。
- 研究者要整區統計、批次匯出可能需電腦端或從後台操作；APP 端以單棵樹木中的時間序列歷史紀錄為主。
